Docker y el mundo de los contenedores: la revolución DevOps

Publicado: 03/11/2025 · Categoría: DevOps

Docker permite empaquetar una aplicación y todas sus dependencias en un contenedor ligero, portátil y reproducible. Esto se traduce en despliegues rápidos y seguros, desde laptops hasta servidores en la nube.

Docker vs Máquinas Virtuales

Máquina Virtual (VM) Contenedor Docker
Incluye sistema operativo completo Comparte kernel del host
Arranque lento Arranque instantáneo
Pesado (GBs) Ligero (MBs)
Menos densidad de apps Muchas instancias por host

¿Cómo funciona Docker?

Docker usa tecnologías del kernel Linux como namespaces y cgroups para aislar procesos:

  • Namespaces: separar vistas del sistema (puertos, procesos, userspace)
  • cgroups: limitar memoria, CPU, I/O
  • UnionFS: imágenes por capas para compartir lo común

Componentes principales

  • Imagen: plantilla inmutable de una app (capas)
  • Contenedor: instancia en ejecución de una imagen
  • Volúmenes: persistencia fuera del contenedor
  • Redes: comunicación interna o externa

Ejemplo de Dockerfile (Node.js)

# Etapa de build FROM node:20-alpine AS build WORKDIR /app COPY package*.json ./ RUN npm ci --only=production COPY . . # Etapa final (más ligera) FROM node:20-slim WORKDIR /app COPY --from=build /app ./ CMD ["node","server.js"]

Multi-stage builds = imágenes más pequeñas y seguras

Buenas prácticas

  • Usar imágenes alpine o distroless para reducir tamaño y superficie de ataque.
  • Incluir un .dockerignore para excluir archivos innecesarios.
  • Fijar versiones de imágenes para garantizar reproducibilidad.
  • Evitar usar root dentro del contenedor.
  • Usar multi-stage para producción.

Comandos esenciales

# Crear imagen docker build -t miapp . # Ejecutar contenedor docker run -p 3000:3000 miapp # Listar contenedores docker ps -a # Ver logs docker logs # Eliminar contenedor/imágenes docker rm ; docker rmi

Seguridad y “distroless”

Las imágenes distroless no incluyen bash ni paquetes innecesarios, reduciendo el vector de ataque. Para apps críticas se recomiendan junto a:

  • Escaneo de imágenes (Snyk, Trivy)
  • Firmas de imágenes (Notary v2, Cosign)
  • Principio de mínimo privilegio

FAQ & Glosario

¿Docker reemplaza las VMs?

No siempre. Las VMs siguen siendo mejores para aislamiento fuerte o sistemas operativos distintos en un mismo host.

¿Qué pasa si borro un contenedor?

Si no usas volúmenes, los datos se pierden. Docker persiste solo lo explícito.

Glosario:

  • DevOps: cultura para entregar software rápido y con calidad.
  • Registry: repositorio de imágenes (Docker Hub, ECR, GCR).
  • Orquestación: gestión de clusters (Kubernetes, Swarm).
